# Workflow Concept

The blog-workflow plugin provides a structured, phase-based approach to content creation. Rather than jumping directly to writing, the workflow guides you through ideation, research, planning, writing, and publishing.

## Why a Structured Workflow?

### Without Structure

```text
"I want to write about Rust"
    ↓
[3 hours of unfocused writing]
    ↓
Half-finished post with unclear focus
```

### With Structure

```text
Idea Phase: "What specifically about Rust? For whom?"
    ↓
Research Phase: "What do I need to learn/verify?"
    ↓
Content Planning: "What sections? What order?"
    ↓
Writing Phase: "Now write with clear direction"
    ↓
Publish Phase: "Review, optimize, ship"
```

## Core Principles

### 1. Separate Concerns

Each phase has a distinct purpose:

- **Idea**: Scope and focus
- **Research**: Knowledge gathering
- **Content**: Structure and outline
- **Post**: Actual writing
- **Publish**: Polish and ship

### 2. Iterative Refinement

Each phase includes:

- **Draft**: Initial attempt
- **Review**: Critical assessment
- **Refine**: Improvement based on review

### 3. Checkpoints Before Proceeding

Don't rush to the next phase. Review checklists ensure quality gates between phases.

### 4. Project-Based Organization

All artifacts for a piece of content live together in a project. This includes ideas, research, outlines, drafts, and feedback.
