### Task 5.1.2: Consolidate SourceSpan Definitions (M1)

**Priority:** Required before Phase 6
**Effort:** 1-2 hours
**Location:** `ir-core/treesitter.py`, `ir-core/models.py`

**Problem:**
Two `SourceSpan` classes with different purposes:

| File | Class | Fields | Purpose |
|------|-------|--------|---------|
| `treesitter.py` | `SourceSpan` | `start_byte`, `end_byte`, `start_point`, `end_point` | Raw tree-sitter data |
| `models.py` | `SourceSpan` | `start_line`, `start_col`, `end_line`, `end_col` | IR serialization |

**Solution:**

1. **Rename** `treesitter.py` class to `TSSourceSpan`:
```python
@dataclass
class TSSourceSpan:
    """Tree-sitter source span with byte offsets and points."""
    file: str
    start_byte: int
    end_byte: int
    start_point: tuple[int, int]  # (row, col)
    end_point: tuple[int, int]

    def to_ir_span(self) -> "SourceSpan":
        """Convert to IR SourceSpan for serialization."""
        from .models import SourceSpan as IRSourceSpan
        return IRSourceSpan(
            file=self.file,
            start_line=self.start_point[0] + 1,  # 1-indexed
            start_col=self.start_point[1],
            end_line=self.end_point[0] + 1,
            end_col=self.end_point[1],
        )
```

2. **Keep** `models.py` class as canonical `SourceSpan`

3. **Update** `TreeNode` to use `TSSourceSpan`:
```python
@dataclass
class TreeNode:
    span: TSSourceSpan  # Changed from SourceSpan
```

4. **Update exports** in `ir-core/__init__.py`:
```python
from .treesitter import TSSourceSpan
from .models import SourceSpan  # Canonical version
```

**Verification:**
- All existing tests pass
- Type checker shows no errors
- Extractor still works

### Task 5.1.5: Improve Executor Security (H1)

**Priority:** Important for production use
**Effort:** 1-2 hours
**Location:** `ir-roundtrip/executor.py`

**Problem:**
`SafeExecutor` uses subprocess isolation but lacks:
- Resource limits (memory, CPU)
- Explicit security documentation
- Network blocking (OS-level)

**Solution:**

1. **Add security disclaimer** to docstring:
```python
"""Safe code execution for round-trip validation.

SECURITY WARNING:
    This executor is designed for TRUSTED code only (e.g., test fixtures).
    It does NOT provide sandbox-level isolation for untrusted code.

    For untrusted code, use container isolation (Docker, gVisor) or
    a dedicated sandbox like Firecracker, Bubblewrap, or nsjail.

Security measures implemented:
    - Subprocess isolation (separate process)
    - Execution timeout (configurable, default 5s)
    - Output size limits (configurable)

NOT implemented (out of scope):
    - Network isolation (use firewall rules)
    - Filesystem isolation (use container)
    - Memory limits (optional, see below)
"""
```

2. **Add optional resource limits** (Linux only):
```python
import resource
import platform

def _set_resource_limits(self) -> None:
    """Set resource limits for child process (Linux only)."""
    if platform.system() != "Linux":
        return

    # Limit memory to 256MB
    memory_limit = self.config.memory_limit_mb * 1024 * 1024
    resource.setrlimit(resource.RLIMIT_AS, (memory_limit, memory_limit))

    # Limit CPU time to timeout + 1 second
    cpu_limit = int(self.config.timeout) + 1
    resource.setrlimit(resource.RLIMIT_CPU, (cpu_limit, cpu_limit))
```

3. **Add to subprocess call**:
```python
result = subprocess.run(
    [self.python_executable, str(temp_path)],
    capture_output=True,
    timeout=actual_timeout,
    text=True,
    preexec_fn=self._set_resource_limits if platform.system() == "Linux" else None,
)
```

4. **Add configuration**:
```python
@dataclass
class ExecutorConfig:
    timeout: float = 5.0
    memory_limit_mb: int = 256
    max_output_bytes: int = 1_000_000
    trusted_mode: bool = True  # Explicit acknowledgment
```

**Verification:**
- Existing tests pass
- Memory-heavy code is killed on Linux
- Documentation clearly states limitations

### Task 5.1.6: Make Python Version Configurable (M5)

**Priority:** Recommended
**Effort:** 1 hour
**Location:** `ir-synthesize-python/generator.py`

**Problem:**
Type annotation generation assumes Python 3.9+ syntax:
```python
# Uses list[int] instead of List[int]
if base in ("list", "dict", "set", "tuple", "frozenset"):
    return f"{base}[{args}]"
```

**Solution:**

1. **Add to SynthConfig**:
```python
@dataclass
class SynthConfig:
    target_version: tuple[int, int] = (3, 10)
    use_future_annotations: bool = True
    # ...
```

2. **Version-aware generation**:
```python
def gen_type_annotation(self, type_ref: TypeRef) -> str:
    if self.config.target_version < (3, 9):
        # Use typing module for older versions
        return self._gen_legacy_annotation(type_ref)
    return self._gen_modern_annotation(type_ref)
```

3. **Add `from __future__ import annotations`** when targeting 3.7+:
```python
def gen_header(self) -> str:
    if self.config.use_future_annotations:
        return "from __future__ import annotations\n\n"
    return ""
```

**Verification:**
- Generated code valid for Python 3.8, 3.9, 3.10, 3.11, 3.12
