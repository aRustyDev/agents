## Findings Summary

### 1. Skills CLI Decision: `npx skills` (Vercel Labs)

| Tool | Downloads | Agents | Status | Verdict |
|------|-----------|--------|--------|---------|
| `npx skills` | **547K/week** | 42 | Active | **✅ Adopt** |
| `npx openskills` | 2.7K/week | ~6 | Active | Consider for AGENTS.md |
| `npx add-skill` | 3.5K/week | 4 | **Deprecated** | ❌ Redirects to skills |

**Rationale:** 200x adoption, Vercel backing, skills.sh registry, comprehensive agent support, `find`/`check`/`update` lifecycle commands.

### 2. Non-Skill Component CLIs

| Component | Dedicated CLI | Distribution Path |
|-----------|--------------|-------------------|
| Commands | None | **Bundle into skills** (Claude Code merged commands into skills) |
| Agents | None | **Bundle into skills** (no dedicated tool) |
| Rules | `prpm` (7K+ packages) | Consider for cross-agent rules |
| Hooks | **None** | **Build opportunity** (largest gap) |
| MCP configs | `skillpm`, `add-mcp` | Partial coverage |

**Key finding:** The `prpm` CLI handles rules, commands, skills, agents, and prompts across IDEs. Worth evaluating as cross-format bridge.

### 3. Prompts Decision: **Do NOT create separate component type**

- Commands are "prompts with slash triggers"
- Rules are "headless prompts" (always injected)
- Skills are "prompts with infrastructure"
- **Recommendation:** Use `type: command | guideline | template | workflow` metadata within SKILL.md
- Remove "Prompts(?)" from Tier 4 component list

### 4. Adopt vs Build Summary

| Action | Tool/Spec | Rationale |
|--------|-----------|-----------|
| **Adopt now** | `npx skills` | Skill distribution standard |
| **Adopt now** | SKILL.md / agentskills.io spec | Critical mass (Anthropic, OpenAI, Microsoft, Google) |
| **Consider** | `@tanstack/intent` | npm-bundled skills (version-pinned) |
| **Consider** | `prpm` | Cross-agent format support |
| **Build** | Hook distribution | No tool exists (largest ecosystem gap) |
| **Build** | Cross-agent rule translation | Different formats per agent |
| **Contribute** | Skill dependency resolution | No spec-level mechanism exists |

### 5. Distribution Gaps (Potential Phase 5)

1. **Hook distribution** - No tool packages or shares hook configurations
2. **Cross-agent rules** - No translation between .cursor/rules/*.mdc, .claude/rules/*.md, .windsurf/rules/*.md
3. **Skill dependencies** - No formal dependency graph at spec level
4. **AGENTS.md for Claude Code** - 3K+ upvotes, no Anthropic response
