# Project Plan Review Checklist

Evaluate plans for structure, scope, alignment with the original idea, and completeness.

## Structure

- [ ] **Research section defined** — Clear scope, sources, and deliverable.
- [ ] **Content deliverables identified** — Know what posts/content will be produced.
- [ ] **Timeline/milestones outlined** — Phases have durations and deliverables.

## Scope

- [ ] **Achievable within stated timeframe** — Timeline is realistic.
- [ ] **Dependencies identified** — External blockers and prerequisites listed.
- [ ] **Risks acknowledged** — Potential issues and mitigations documented.

## Alignment

- [ ] **Aligns with original idea** — Plan serves the idea's stated goals.
- [ ] **Addresses target audience needs** — Deliverables match audience expectations.
- [ ] **Clear success criteria** — Know when the project is complete.

## Completeness

- [ ] **All sections filled** — No empty or placeholder sections.
- [ ] **No placeholder content remaining** — All `{{placeholders}}` replaced.
- [ ] **Links to idea artifact present** — `parent` field points to idea.md.

## Severity Levels

| Level | Meaning | Action |
|-------|---------|--------|
| **fail** | Blocks approval | Must fix before proceeding |
| **warn** | Quality concern | Should address, but can proceed |
| **pass** | Meets criterion | No action needed |

## Fail Criteria (self-review catches these)

- Research section missing or empty
- No content deliverables listed
- No timeline provided
- Parent link to idea.md missing

## Warn Criteria (human review catches these)

- Timeline seems aggressive
- Risks section is minimal
- Success criteria are vague
