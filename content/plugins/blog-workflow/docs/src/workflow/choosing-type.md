# Choosing a Post Type

Use this guide to select the appropriate post type for your content.

## Decision Tree

### Start Here

**What are you doing?**

```text
Teaching/Explaining something
├── Do you know the topic well?
│   ├── Yes → Is it a simple concept?
│   │   ├── Yes → Do you want to use metaphors?
│   │   │   ├── Yes → ELI5
│   │   │   └── No → Tutorial
│   │   └── No → Do you want progressive complexity?
│   │       ├── Yes → 5 Levels
│   │       └── No → Tutorial or Deep Dive
│   └── No → Study Review (share what you learned)
│
Showing a process
├── Is it reproducible steps?
│   ├── Yes → Walk-through
│   └── No → Is it learning/studying?
│       ├── Yes → Study Review
│       └── No → Updates
│
Evaluating something
├── Is it comparing alternatives?
│   ├── Yes → Comparison
│   └── No → Review
│
Sharing opinion/wisdom
├── Is it grounded in experience?
│   ├── Yes → Is it evergreen advice?
│   │   ├── Yes → Dev Blog
│   │   └── No → Opinions
│   └── No → Opinions
│
Documenting research
├── Is it formal with methodology?
│   ├── Yes → Research
│   └── No → Deep Dive or Study Review
│
Project documentation
├── When in the project?
│   ├── Before starting → Planning
│   ├── During → Updates
│   └── After completion → Retrospective
│
Exploring ideas
├── Is it a known problem?
│   ├── Yes → Problems
│   └── No → Open Questions
│
Announcing something
└── Announcement

Curating resources
└── Reference
```

| "I want to..." | Type |
|----------------|------|
| Teach a concept step-by-step | Tutorial |
| Explain something simply | ELI5 |
| Show progressive complexity | 5 Levels |
| Document my learning process | Walk-through |
| Share what I learned | Study Review |
| Evaluate a tool/course/book | Review |
| Compare A vs B | Comparison |
| Go deep on a technical topic | Deep Dive |
| Present formal findings | Research |
| State a known problem | Problems |
| Explore a possibility | Open Questions |
| Document pre-project thinking | Planning |
| Share project progress | Updates |
| Reflect on completed work | Retrospective |
| Share my opinion | Opinions |
| Give experienced advice | Dev Blog |
| Announce something new | Announcement |
| Curate resources | Reference |

## Type Comparison

### Teaching Types

| Type | You Know | Audience | Depth |
|------|----------|----------|-------|
| Tutorial | A lot | Learners | Complete |
| ELI5 | A lot | Beginners | Simplified |
| 5 Levels | A lot | All levels | Progressive |
| Study Review | Some | Peers | Shared learning |

**Choose Tutorial** when: You can teach the complete topic.
**Choose ELI5** when: The topic is complex but you can simplify it.
**Choose 5 Levels** when: You want to serve multiple audience levels.
**Choose Study Review** when: You're sharing your learning, not teaching.

### Process Types

| Type | Known Outcome | Reproducible | Voice |
|------|---------------|--------------|-------|
| Tutorial | Yes | Yes | Authoritative |
| Walk-through | Maybe | Yes | Exploratory |
| Study Review | No | No | Reflective |

**Choose Tutorial** when: You know the steps and outcome.
**Choose Walk-through** when: You're documenting discovery.
**Choose Study Review** when: You're reflecting on learning.

### Analysis Types

| Type | Subject | Scope | Formality |
|------|---------|-------|-----------|
| Deep Dive | Technical topic | Narrow | Industry |
| Research | Any | Narrow | Academic |
| Review | External thing | Single item | Evaluative |
| Comparison | Multiple items | Comparative | Evaluative |

**Choose Deep Dive** when: Industry-level technical depth.
**Choose Research** when: Formal investigation with methodology.
**Choose Review** when: Evaluating one external thing.
**Choose Comparison** when: Comparing multiple options.

### Project Types

| Type | When | Purpose |
|------|------|---------|
| Planning | Before | Document goals and approach |
| Updates | During | Progress diary |
| Retrospective | After | Lessons learned |

These form a natural project-arc relationship.

## Common Mistakes

### Wrong: Tutorial when you're learning

If you're learning while writing, use Walk-through or Study Review.
Tutorial implies expertise.

### Wrong: Deep Dive for opinions

Deep Dive is analytical, not opinionated.
Use Dev Blog or Opinions for personal takes.

### Wrong: Research for informal exploration

Research implies methodology and rigor.
Use Study Review or Open Questions for exploration.

### Wrong: Review for multiple items

Review evaluates one thing.
Use Comparison for A vs B analysis.

## When Types Blend

Sometimes content spans types. Choose the primary type:

| Content | Primary Type | Secondary Aspect |
|---------|--------------|------------------|
| Tutorial with opinions | Tutorial | Add "My take" section |
| Walk-through with teaching | Walk-through | Explain as you go |
| Research with recommendations | Research | Conclusions section |
| Comparison with deep analysis | Comparison | Detailed criteria |

If truly split, consider multiple posts in a series.
