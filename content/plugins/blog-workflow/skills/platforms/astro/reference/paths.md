# AstroPaper Directory Conventions

File locations and naming conventions for AstroPaper projects.

## Content Locations

| Purpose | Path | Notes |
|---------|------|-------|
| Published posts | `src/data/blog/` | Final destination for live content |
| Build output | `dist/` | Generated by `astro build` |
| Drafts (workflow) | `content/_drafts/` | Managed by blog-workflow plugin |
| Projects (workflow) | `content/_projects/` | Multi-post project directories |
| Templates (workflow) | `content/_templates/` | Copied by `/blog/init` |

## Promotion Path

```
content/_projects/<name>/drafts/final.md
    ↓ (move to drafts/)
content/_drafts/<slug>.md
    ↓ (promote via /blog/publish/promote)
src/data/blog/<slug>.md
```

## File Naming

- Post files: `<slug>.md` (lowercase, hyphenated)
- Slug format: lowercase, hyphens only, max 60 characters
- No special characters except hyphens

## AstroPaper Project Structure

Typical AstroPaper project layout:

```
project-root/
├── astro.config.mjs          # Astro configuration
├── src/
│   ├── data/
│   │   └── blog/             # Published posts
│   │       ├── post-one.md
│   │       └── post-two.md
│   ├── content/
│   │   └── config.ts         # Content collection config
│   ├── layouts/
│   ├── pages/
│   └── components/
├── dist/                      # Build output (gitignored)
├── content/                   # Blog workflow directories
│   ├── _drafts/
│   ├── _projects/
│   └── _templates/
└── package.json
```

## Hook Paths

The blog-workflow hooks reference these Astro-specific paths:

| Hook | Watches | Purpose |
|------|---------|---------|
| `validate-blog-frontmatter.sh` | `content/_drafts/*`, `content/_projects/*` | Validate frontmatter on edit |
| `promote-safety.sh` | `src/data/blog/*` | Prevent accidental writes to published dir |

## Notes

- `src/data/blog/` is the AstroPaper convention. Some older AstroPaper versions use `src/content/blog/`.
- The `content/` directory at project root is used by the blog-workflow plugin, not by Astro itself.
- `dist/` is typically gitignored and regenerated on each build.
