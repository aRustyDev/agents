# Issue Templates

GitHub issue body templates for skill validation findings.

## Template Selection

| Severity | Issue Type | Labels |
|----------|------------|--------|
| Critical | `fix` | `bug`, `skills`, `priority:high` |
| Warning | `enhance` | `enhancement`, `skills` |
| Suggestion | `docs` | `documentation`, `skills`, `good first issue` |

## Title Format

```
<type>(skills): <skill-name> <brief-description>
```

### Examples

| Severity | Title |
|----------|-------|
| Critical | `fix(skills): lang-rust-dev exceeds 800 line budget` |
| Warning | `enhance(skills): convert-python-rust add navigation section` |
| Suggestion | `docs(skills): meta-convert-dev improve trigger words` |

## Critical Issue Template

Use for issues that block functionality or violate must-fix criteria.

```markdown
## Summary

The skill `<skill-name>` has a critical issue that must be fixed.

## Issue

**Type:** <issue-type>
**File:** `<file-path>`
**Line(s):** <line-numbers if applicable>

<detailed description of the issue>

## Expected

<what the skill should have>

## Current

<what the skill currently has>

## Suggested Fix

<specific, actionable fix>

## Acceptance Criteria

- [ ] Issue is resolved
- [ ] Skill passes `/refine-skill --check-only`
- [ ] No new issues introduced

## Context

- Skill path: `<skill-path>`
- Detected by: `<command or recipe>`
- Best practice reference: [Skills Best Practices](https://docs.anthropic.com/en/docs/agents-and-tools/claude-code/skills)

---
Generated by validation command
```

## Warning Issue Template

Use for issues that impact quality but don't block functionality.

```markdown
## Summary

Improvement opportunity for skill `<skill-name>`.

## Current State

<description of current state>

## Proposed Improvement

<specific improvement>

## Benefits

- <benefit 1>
- <benefit 2>

## Implementation Notes

<any relevant implementation details>

## Acceptance Criteria

- [ ] Improvement implemented
- [ ] Skill quality improved
- [ ] No regressions

## Context

- Skill path: `<skill-path>`
- Detected by: `<command or recipe>`
```

## Suggestion Issue Template

Use for nice-to-have improvements and documentation enhancements.

```markdown
## Summary

Documentation improvement for skill `<skill-name>`.

## Suggestion

<description of suggestion>

## Rationale

<why this would be helpful>

## Examples

<examples of the improvement, if applicable>

## Context

- Skill path: `<skill-path>`
- Detected by: `<command or recipe>`
```

## Umbrella Issue Template

Use when 7+ findings exist for a single skill. Create individual issues only for critical items.

```markdown
## Summary

Skill `<skill-name>` requires refinement to meet best practices.

## Findings

### Critical (must fix)

- [ ] #<issue-num> - <summary>
- [ ] #<issue-num> - <summary>

### Warnings (should fix)

- [ ] <description>
- [ ] <description>

### Suggestions (nice to have)

- [ ] <description>
- [ ] <description>

## Analysis Report

<embed full analysis report>

## Progress

| Category | Total | Fixed | Remaining |
|----------|-------|-------|-----------|
| Critical | X | 0 | X |
| Warning | X | 0 | X |
| Suggestion | X | 0 | X |

## Related

- Best practices: [Skills Best Practices](https://docs.anthropic.com/en/docs/agents-and-tools/claude-code/skills)
- Validation skill: `meta-skill-validation-dev`

---
Generated by validation command
```

## Batch Strategy

| Findings Count | Strategy |
|----------------|----------|
| 1-3 | Create individual issues |
| 4-6 | Create individual issues, add cross-references |
| 7+ | Create umbrella issue, individual only for critical |

### Cross-Reference Format

When creating related issues, include:

```markdown
## Related Issues

- #<num> - <related issue title>
- Part of skill refinement for `<skill-name>`
```

## Creating Issues

### Using gh CLI

```bash
gh issue create \
  --repo <owner>/<repo> \
  --title "<type>(skills): <skill-name> <brief-description>" \
  --body "<issue-body>" \
  --label "skills,<severity-label>"
```

### Using GitHub MCP

```
mcp__github__issue_write with method: "create"
```

## Validation Command Integration

Commands that create issues:

| Command | Flag | Creates Issues |
|---------|------|----------------|
| `/refine-skill` | `--create-issues` | Yes |
| `/validate-lang-conversion-skill` | Default | Yes |
